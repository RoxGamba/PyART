.. py:class:: CoordsChange

   Collection of coordinate transformations with unified symbolic and
   numeric interfaces.

   Provides both numerical and symbolic (sympy) APIs for standard
   coordinate transformations used in Post-Newtonian and EOB/ADM context.
   Each transformation returns AnalyticExpression objects or plain numeric
   arrays depending on the calling method.

   Supported transformations:
   - Polar <-> Cartesian  (2D phase-space)
   - EOB <-> ADM coordinates/momenta

   References for EOB <-> ADM formulas:
   - Buonanno & Damour (1998) arXiv:gr-qc/9811091 (Appendix E)
   - Bini & Damour (2012) arXiv:1210.28...

      Validate the PN order supported by the coordinate maps.

      :param PN_order: Requested post-Newtonian truncation order.
      :type PN_order: int

      :returns: Validation succeeds silently when the order is supported.
      :rtype: None

      :raises ValueError: If ``PN_order`` is not one of ``0``, ``1``, or ``2``.



   .. py:method:: Eob2Adm(qe_vec, pe_vec, nu, PN_order=2)
      :staticmethod:


      Transform EOB phase-space coordinates into ADM coordinates.

      :param qe_vec: EOB position vector.
      :type qe_vec: sequence
      :param pe_vec: EOB momentum vector.
      :type pe_vec: sequence
      :param nu: Symmetric mass ratio.
      :type nu: float or sympy.Expr
      :param PN_order: Post-Newtonian order retained in the transformation. Supported
                       values are ``0``, ``1``, and ``2``.
      :type PN_order: int, optional

      :returns: Pair ``(Q_ADM, P_ADM)`` as NumPy arrays for numeric input or lists
                of :class:`AnalyticExpression` objects for symbolic input.
      :rtype: tuple

      :raises ValueError: If ``PN_order`` is not supported.

.. py:function:: eob_ID_to_ADM(eob_Wave, verbose=False, PN_order=2, rotate_on_x_axis=True)

   Convert EOB initial data into ADM puncture-style initial data.

   :param eob_Wave: Waveform-like object with ``pars`` and ``dyn`` mappings and a
                    ``get_Pr()`` method. The object must provide ``pars["q"]``, the first
                    entries of ``dyn["r"]``, ``dyn["phi"]``, ``dyn["Pphi"]``, and the
                    first radial momentum from ``get_Pr()``.
   :type eob_Wave: object
   :param verbose: If ``True``, print a diagnostic summary of the transformed data.
   :type verbose: bool, optional
   :param PN_order: Post-Newtonian order retained in the EOB-to-ADM transformation.
   :type PN_order: int, optional
   :param rotate_on_x_axis: If ``True``, rotate the ADM coordinates so the punctures lie on the
                            ``x`` axis at the initial time.
   :type rotate_on_x_axis: bool, optional

   :returns: Dictionary containing Cartesian coordinates and momenta in both EOB
             and ADM frames, plus puncture placement information.
   :rtype: dict[str, object]

   :raises ValueError: If the transformed ADM separation is zero or the PN order is invalid.
