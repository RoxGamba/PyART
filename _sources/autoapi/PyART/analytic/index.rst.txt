   .. py:method:: truncate(var, max_order)

      Truncate the expression by removing terms whose exponent in ``var``
      is strictly greater than ``max_order``.

      When the expression is already a sum of termwise powers in the target
      variable, use cached exponent filtering. Fall back to SymPy series
      expansion plus the same exponent filtering for non-termwise
      expressions.

      In the fast path, exact ``log(var)`` factors are temporarily replaced
      by a placeholder symbol before exponent extraction. This makes
      logarithms behave like coefficients for power counting, so terms such
      as ``x**2 * log(x)`` and ``x**3 * log(x)**2`` are treated as orders
      ``2`` and ``3`` respectively. More complicated logarithmic dependence,
      such as ``log(1 + x)`` or ``log(x**2)``, is not normalized this way and
      will typically force the slower series-based fallback.

      :param var: Variable used to define the truncation order.
      :type var: str or sympy.Symbol
      :param max_order: Maximum power of ``var`` to retain.
      :type max_order: object

      :returns: Truncated analytic expression.
      :rtype: AnalyticExpression

      :raises ValueError: If no expression is stored.

.. py:class:: MathematicaParser

   Parse Mathematica source text and convert symbolic forms to SymPy.

   The parser handles common source normalization steps, extracts metadata
   and assignments from Mathematica files, and converts special symbolic
   constructs such as ``SeriesData`` into explicit SymPy expressions.

   The class is intentionally generic. Library-specific naming conventions
   and identifier rewrites should stay with the caller and be applied through
   the helper methods exposed here.


   .. py:method:: normalize_source(text: str) -> str

      Normalize raw Mathematica source before structural parsing.

      :param text: Raw Mathematica source text.
      :type text: str

      :returns: Source with comments removed, box expressions replaced, and common
                escaped identifiers simplified.
      :rtype: str



   .. py:method:: _is_escaped_by_backslash(text: str, index: int) -> bool

      Return whether ``text[index]`` is escaped by backslashes.

   .. py:method:: get_entry(name=None, path=None)

      Retrieve a PN quantity from PNPedia.

      The PNPedia structure (inside Core post-Newtonian quantities) is:
      - orbit type (Circular, Elliptic, Hyperbolic)
      - Spin (NonSpinning, Spinning)
      - Precession (Precessing, Nonprecessing)
      - Tides (With tidal effects, Without tidal effects)
      - Quantity (
        Waveform, Fluxes, Constants of motion, Waveform frequencies):
          - Constants of motion (Energy, Angular momentum)
          - Fluxes (Angular momentum flux, Energy flux)
          - Waveform (h_l_m.txt)

      Quantities are stored as Mathematica-readable text files.
      They are converted into SymPy expressions on demand.

      Inside each folder is also a README.md providing:
      - arXiv references/sources
      - Notation & definitions
      - Endorser

      :param name: The name of the PN quantity to retrieve,
      :type name: str
      :param path: The path to the PNPedia file to be loaded. If None, it is resolved
                   from the indexed quantity name.
      :type path: str, optional

      :returns: Parsed and cached representation of the requested quantity.
      :rtype: PNPediaEntry

.. py:class:: CoordsChange

   Collection of coordinate transformations with unified symbolic and
   numeric interfaces.

   Provides both numerical and symbolic (sympy) APIs for standard
   coordinate transformations used in Post-Newtonian and EOB/ADM context.
   Each transformation returns AnalyticExpression objects or plain numeric
   arrays depending on the calling method.

   Supported transformations:
   - Polar <-> Cartesian  (2D phase-space)
   - EOB <-> ADM coordinates/momenta

   References for EOB <-> ADM formulas:
   - Buonanno & Damour (1998) arXiv:gr-qc/9811091 (Appendix E)
   - Bini & Damour (2012) arXiv:1210.28...

.. py:function:: eob_ID_to_ADM(eob_Wave, verbose=False, PN_order=2, rotate_on_x_axis=True)

   Convert EOB initial data into ADM puncture-style initial data.

   :param eob_Wave: Waveform-like object with ``pars`` and ``dyn`` mappings and a
                    ``get_Pr()`` method. The object must provide ``pars["q"]``, the first
                    entries of ``dyn["r"]``, ``dyn["phi"]``, ``dyn["Pphi"]``, and the
                    first radial momentum from ``get_Pr()``.
   :type eob_Wave: object
   :param verbose: If ``True``, print a diagnostic summary of the transformed data.
   :type verbose: bool, optional
   :param PN_order: Post-Newtonian order retained in the EOB-to-ADM transformation.
   :type PN_order: int, optional
   :param rotate_on_x_axis: If ``True``, rotate the ADM coordinates so the punctures lie on the
                            ``x`` axis at the initial time.
   :type rotate_on_x_axis: bool, optional

   :returns: Dictionary containing Cartesian coordinates and momenta in both EOB
             and ADM frames, plus puncture placement information.
   :rtype: dict[str, object]

   :raises ValueError: If the transformed ADM separation is zero or the PN order is invalid.
